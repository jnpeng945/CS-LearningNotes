2. `int *p = new int[40](); ` 在分配空间后，对每个int进行默认初始化。即p[n]的值（n的取值范围是0-39）都是0。

   `int *p = new int[40](0);`  语法错误，正确的是用大括号 `int *p = new int[40]{0,1,2,3,4};` 

3. 局部变量可以和全局变量重名，编译的时候不会出现错误。此时，局部变量会屏蔽全局变量。要用全局变量，需要**使用 " :: "** （域解析符）。局部变量可以与全局变量同名，在函数内引用这个变量时，会用到同名的局部变量，而不会用到全局变量。

   ```cpp
   #include<iostream>
   using namespace std;
   int a;      //定义全局变量
   int main() {
       int a = 3;      //定义局部变量并赋值
       cout << "a = " << a << endl;
       ::a = 2;        //给全局变量赋值
       cout << "a = " << ::a << endl;
   }
   ```

5. 多态类中的虚函数表建立在**编译阶段**，编译器对每个包含虚函数的类创建一个虚函数表，在虚函数表中，放置这个类的虚函数地址。

6. 除了类属关系运算符 `.` 、成员指针运算符 `.*` 、作用域运算符`::` 、sizeof 运算符和三目运算符 `?:` 之外，C++中的所有运算符都可以重载。

7. `const `和 `#define` 定义常量的区别：

   `define`宏是在预处理阶段展开。`const`常量是编译运行阶段使用；

   宏没有类型，不做任何类型检查，仅仅是展开。`const` 常量有具体的类型，在编译阶段会执行类型检查。

   `define` 宏仅仅是展开，有多少地方使用，就展开多少次，不会分配内存。`const` 常量会在内存中分配（可以是堆中也可以是栈中）。

7. 宏定义需要定义多行代码时，在行尾放一个 `\ ` ，编译器会忽略行尾的换行符，起到续行的作用。

8. 继承

   公有继承时基类中的 public 成员在派生类中仍是 public；

   公有继承时基类中的 protected 成员在派生类中仍是 protected；

   私有继承时基类中的 public 成员在派生类中是 private。

10. 构造函数

    - 构造函数可以被重载
    - 构造函数可以是内联函数
    - 构造函数可以没有参数
11. 面向对象的理解
    - 面向对象的最重要的特性是支持继承、封装和多态
    - 系统设计应该遵循开闭原则，系统应该稳定不可被修改，但应支持通过继承、组合等方式进行扩展
    - 面向对象设计时，每个类的职责应该单一，不要在一个类中引入过多的接口
    - 过程式语言和面向对象的语言各有优势，过程式语言更加灵活，面向对象语言更加强调抽象和封装

1. 电子邮件收发流程中发件人发来的电子邮件保存在收件人的电子邮箱中，收件人可随时接收。

2. 物理层常用设备：中继器、集线器；数据链路层常用设备：网桥、交换机；网络层常用设备：路由；传输层常用设备：网关。划分子网——网络层利用路由器实现。

3. ARP协议

   - 从已知的IP地址找出对应物理地址的映射过程叫做**地址解析**；
   - 从已知的物理地址找出对应IP地址的映射过程叫做**反向地址解析**；
   - ARP定义了请求分组与应答分组的格式；
   - 请求分组：ARP进程在局域网中广播一个ARP请求分组（广播）；
   - 应答分组：向发送ARP分组的主机发送一个响应分组（单播）；

4. 操作系统的一些特别端口要为特定的服务做预留，必须要root权限才能打开的端口：所有小于1024的每个端口。Unix系统有保留端口 (reserved port) 的概念，指的是小于1024的任何端口。这些端口只能赋予特权用户进程的套接字。

5. Socket 用于描述**IP地址和端口**，是一个通信链的句柄；**Socket 客户端**的端口是不固定的；**Socket服务端**的端口是固定的。IPV4 下，两种基本的通信方式分别是 TCP 和 UDP，前者是面向连接的可靠的字节流服务，通信之前必须要先建立起 Socket 连接，而后者是面向无连接的数据包服务，通信之前无需建立起任何连接。所以 **Socket 可以基于TCP 面向连接，也可以基于UDP无连接**。

6. DNS 何时使用 TCP 协议，何时使用 UDP 协议？

   DNS（端口53）同时占用UDP和TCP是公认的，DNS在进行区域传输或者响应报文超过512字节的时候使用TCP协议，其它时候则使用UDP协议。

7. DNS 为什么既使用 TCP 又使用 UDP？

   UDP 报文的最大长度为 512 字节，而 TCP 则允许报文长度超过 512 字节。**当 DNS 查询超过 512 字节时，协议的 TC 标志出现删除标志**，这时则使用 TCP 发送。通常传统的 UDP 报文一般不会大于 512 字节。 

8. DNS 区域传输：DNS 的规范规定了2种类型的 DNS 服务器，一个叫主 DNS 服务器，一个叫辅助 DNS 服务器。在一个区中主DNS服务器从自己本机的数据文件中读取该区的DNS数据信息，而辅助DNS服务器则从区的主DNS服务器中读取该区的DNS数据信息。当一个辅助DNS服务器启动时，它需要与主DNS服务器通信，并加载数据信息，这就叫做区域传输（zone transfer）。

10. HTTP协议

   HTTP 协议是应用层协议，它的底层基于 TCP 连接；HTTP协议规定了浏览器与服务器通信的四个步骤，依次是：建立连接、发送请求、接收响应、关闭连接；HTTP请求报文中，包含请求方式、请求路径、协议版本、消息头等内容；HTTP响应报文中，包含状态码、状态名、协议版本、消息头等内容。

12. DHCP 协议

    dhcp 的配置文件是 /etc/dhcpd.conf，通常包括三部分：

    - parameters，表明如何执行任务，是否要执行任务，或将哪些网络配置选项发送给客户

      如 fixed-address ip：分配给客户端一个固定的地址

    - declarations，用来描述网络布局、提供客户的 IP 地址等

      如 host 主机名称：参考特别的主机

      range 起始 IP 终止 IP：提供动态分配 IP 的范围

    - option，用来配置 DHCP 可选参数，全部用 option 关键字作为开始。

      命令：ip dhcp pool *dhcp-pool* 配置地址池名并进入地址池配置模式

13. IPV6

    IPV6 允许在报头后增加扩展报头，以扩展新的功能；IPV6的安全机制是基于IPSEC的协议实现的；IPV6的路由选择是基于地址前缀概念实现的，便于建立层次化的路由结构；**IPV6的安全机制是必须的**，非可选。

# 3 操作系统

1. 进程：一个进程包括控制结构和执行结构；控制结构是进程控制块PCB，执行结构包括程序以及需要操纵的数据集合。

2. 用于进程控制的原语：进程创建；进程撤销；进程阻塞；进程唤醒。

   **创建原语**：创建一个就绪状态的进程，使进程从创建状态变迁为就绪状态；**撤销原语**：使进程从执行状态变迁为完成状态；**阻塞原语**：使进程从运行状态变迁为阻塞状态；**唤醒原语**：使进程从阻塞状态变迁为就绪状态。

3. 线程和进程

   进程间相互独立，同一进程的各线程间共享；某进程内的线程在其它进程不可见；使用 start() 方法可以使一个线程成为可运行的，但是它不一定立即开始运行；<u>线程上下文</u>切换比<u>进程上下文</u>切换要快得多；当一个线程因为抢先机制而停止运行，它被放在可运行队列的后面。
